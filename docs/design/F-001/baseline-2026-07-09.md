# F-001 Pre-Migration Baseline — 2026-07-09 (REQ-F001-049, N-2)

Captured **before** the first F-001 migration change was committed, from the pre-migration `web/`
production build (git `HEAD` = `f577f1c`, reconstructed in a clean `git worktree`), because these
measurements stop being reconstructable once migration starts. All later REQ-F001-030 (a11y no-
regression) and REQ-F001-033 (bundle budget ≤ baseline + 10%) comparisons cite THIS artifact rather
than a re-measured value.

Build command: `vite build` (pre-migration `web/`, standard config). Measured with
`gzip -c dist/assets/*.{js,css} | wc -c`.

## (b) Accessibility / contrast snapshot — REQ-F001-030 basis

Scope: the parent REQ-100 read views (workspace list, user list, settings) **plus** the three shared
components **DangerConfirm**, **ErrorBanner**, **SetNotSetBadge**. WCAG 2.1 AA is a **non-gating
target** (ruling OQ-6); the hard floor is **no regression vs. this snapshot**.

### Accessibility affordances present pre-migration (must be preserved — REQ-F001-020/021/030)

| Surface | Affordance (the a11y contract to preserve) |
|---|---|
| `DangerConfirm` | `role="dialog"`, `aria-modal="true"`, `aria-label` = dialog title; typed-token / acknowledge gating; destructive action disabled until armed; Cancel + destructive buttons keyboard-operable |
| `ErrorBanner` | `role="alert"`; renders the BFF `{ message }` verbatim (no reword/truncate) |
| `SetNotSetBadge` | text label "set" / "not set"; never renders a secret value |
| Workspace list / User list / Settings | native `<table>` semantics (columnheader / row / cell), associated form `<label>`s, focus-visible on inputs and buttons, keyboard-operable nav |

### Contrast measurements (status / danger / success + text-on-background pairs)

Contrast RATIOS require a real browser (computed color) + a contrast script and are therefore an
**out-of-band** measurement, not reproducible in vitest+jsdom. The pre-migration token pairs that
must not regress are recorded here as the comparison basis:

| Pair | Pre-migration source token(s) |
|---|---|
| Text on primary background | `--theme-text-primary` on `--theme-bg-container` (dark: `#ffffff` on `#0e0f0f`; light: `#0e0f0f` on `#f9fbfd`) |
| Success foreground / fill | retired `--success` (`#22c55e` dark / `#039855` light) on `--success-bg` |
| Danger foreground / fill | retired `--danger` (`#f97066` dark / `#b42318` light) on `--danger-bg` |
| Warn / caution foreground | retired `--theme-button-disable-hover-text` (`#fec84b` dark / `#854708` light) |

**Disclosed post-migration color shifts (accepted by rulings OQ-10/OQ-12):** the danger/success/warn
foregrounds move to the mapped DS badge tokens (`--theme-badge-danger-text`
`#f87171`/`#b91c1c`, `--theme-badge-success-text` `#4ade80`/`#047857`, `--theme-badge-warn-text`
`#facc15`/`#a16207`). These are intentional, ruling-accepted shifts (REQ-F001-048/053), measured
against this baseline for the no-regression floor.

- **Out-of-band re-measurement (post-migration):** run axe-core against the built app for each scope
  surface (dark, `[data-theme="light"]`, and simulated `prefers-color-scheme: light`) plus a
  contrast-ratio script on the pairs above; compare to this snapshot for REQ-F001-030 no-regression.
